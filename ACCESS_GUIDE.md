# 🌐 博客访问指南

## 🎯 重要提示

将军，服务器上所有服务都运行正常！问题是**云服务器安全组**没有开放端口 3000 和 3001。

## ✅ 服务运行状态

| 服务 | 本地访问 | 外部访问 | 说明 |
|------|---------|---------|------|
| **后端 API** | ✅ http://localhost:3001 | ❌ 端口未开放 | 需要开放安全组 |
| **前端网站** | ✅ http://localhost:3000 | ❌ 端口未开放 | 需要开放安全组 |
| **Nginx (后端)** | ✅ http://localhost:8080 | ⚠️ 需测试 | 通过 Nginx 代理 |
| **Nginx (前端)** | ✅ http://localhost:80 | ⚠️ 需测试 | 通过 Nginx 代理 |

## 🔧 解决方案

### 方案 A：配置云服务器安全组（推荐）

1. **登录云服务器控制台**
   - 登录你的云服务商（腾讯云/阿里云等）
   - 找到服务器：147.139.183.210

2. **配置安全组规则**
   - 添加入站规则：
     - 端口：3000，协议：TCP，来源：0.0.0.0/0
     - 端口：3001，协议：TCP，来源：0.0.0.0/0
     - 或者直接开放：端口 80、443、3000、3001

3. **保存并应用规则**

4. **等待 1-2 分钟**后访问

### 方案 B：使用 Nginx 端口（临时方案）

如果安全组已经开放了 80 或 8080 端口，可以直接访问：

- **前端**：http://147.139.183.210 （通过 Nginx 端口 80）
- **后端 API**：http://147.139.183.210:8080 （通过 Nginx 端口 8080）

## 📝 云服务商安全组配置示例

### 腾讯云
1. 登录：https://console.cloud.tencent.com/cvm/instance
2. 找到服务器 147.139.183.210
3. 点击"安全组" → "配置规则"
4. 添加入站规则：
   - 类型：自定义
   - 来源：0.0.0.0/0
   - 协议端口：TCP:3000
   - 策略：允许
5. 重复添加端口 3001
6. 保存

### 阿里云
1. 登录：https://ecs.console.aliyun.com/
2. 找到实例 147.139.183.210
3. 点击"安全组" → "配置规则"
4. 添加入站规则：
   - 授权策略：允许
   - 协议类型：自定义 TCP
   - 端口范围：3000/3001
   - 授权对象：0.0.0.0/0
5. 保存

## 🆘 如果还是无法访问

1. **检查安全组配置**
   - 确认端口已添加
   - 确认来源是 0.0.0.0/0
   - 确认策略是"允许"

2. **检查服务器防火墙**
   ```bash
   iptables -L -n
   ```

3. **检查 Nginx 配置**
   ```bash
   nginx -t
   systemctl status nginx
   ```

4. **查看 PM2 日志**
   ```bash
   pm2 logs
   ```
